# Security Policy

Please report suspected vulnerabilities privately to
alexisbeaulieu97@gmail.com. Do not open a public GitHub issue for a
vulnerability report.

Include as much of the following as you can:

- Repository name and version, tag, or commit.
- Affected command, workflow, or API.
- Security impact and who can trigger it.
- Reproduction steps or a minimal proof of concept.
- Relevant logs, output, or config snippets with secrets and private data
  removed.

Reports are handled on a best-effort basis. I will try to acknowledge useful
reports, clarify impact, and remediate or document mitigations when practical.
